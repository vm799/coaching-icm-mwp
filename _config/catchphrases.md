## Instructions for Script Writers

When writing Stage 03 scripts (enterprise angle):

1. **Include 2-3 soundbites minimum** in every enterprise script
2. **Mark them clearly:** `[SOUNDBITE: "exact phrase"]` in the script
3. **Use these anchors:**
   - Open with a problem soundbite (hook exec attention)
   - Middle with a solution soundbite (memorable takeaway)
   - Close with an action soundbite (what to do Monday)
4. **Test readability:** Soundbite should read naturally in script context (not inserted randomly)
5. **Check quotability:** Would an exec repeat this? Tweet it? Use in a board deck?

## Tone Guardrails (Keep Catchphrases...)

- **Confident, not arrogant** ("This works" not "This is obviously right")
- **Data-grounded, not aspirational** ("Reduces latency 40%" not "Makes everything fast")
- **Actionable, not abstract** ("Get decision structure right before scale" not "Think strategically")
- **Memorable, not clever** ("Tokens are attention" not "Context is like a caffeinated squirrel")
- **Repeatable, not one-liner jokes** (Execs will say it in meetings; make it sound intelligent)
