## Deck Metadata

- **Total slides:** 10-12 (adjust to fit 30 min talk)
- **Timing per slide:** 2-3 min typical (some faster: title, CTA; some longer: examples)
- **Design notes:**
  - Use consistent color scheme (teaching stream: [color], enterprise stream: [color])
  - Keep text minimal (6 words per slide max, exception: code snippets)
  - Use white space (slides feel crowded if busy)
  - Icons/diagrams > photos (easier to animate, timeless)

- **Accessibility:**
  - [ ] Text readable at distance (test at 8px min for projector)
  - [ ] Color not only differentiator (alt: pattern or icon)
  - [ ] Alt text for diagrams (for screen readers)

---

## Instructions for Designers

**Input:** Deck outline (this template)

**Output:** PowerPoint / Keynote / Google Slides deck with:
1. All text from "Text" sections
2. Visuals per "Visual" description
3. Speaker notes from "Speaker notes" sections

**Reference script:** Use script excerpt from "From script" to understand context (what's the speaker saying when this slide appears?)

**Tone:**
- Teaching stream: warm, colorful, learning-focused icons
- Enterprise stream: professional, data-forward, business graphics

---

## Pro Tips

- **Transitions:** Minimal (none or fade). Distraction-free.
- **Animations:** Text appears with speaker, not auto-advance. Speaker controls pacing.
- **Visuals:**
  - Avoid clipart. Use icons or real screenshots.
  - Color code if helpful (e.g., "Teaching concept" = blue, "Risk" = red)
  - Diagrams > photos > clipart
- **Reading order:** Top-left to bottom-right. Test readability.
